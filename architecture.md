## 2. 실전 인프라 트러블슈팅 일지 (Troubleshooting Log)

### 🔥 Issue 1: DirectoryLoader 패키지 의존성 누수 및 자원 낭비
* **현상:** 다중 가이드라인 문서 자동 인덱싱을 위해 LangChain의 `DirectoryLoader`를 호출했으나 백엔드에서 무거운 `unstructured` 패키지 누수 에러(`ModuleNotFoundError: No module named 'unstructured'`) 발생.
* **원인 분석:** `DirectoryLoader`는 기본값으로 강력하지만 매우 무겁고 복잡한 파싱 라이브러리를 요구함. GPU가 없는 제한된 리눅스 가상 서버 환경에 이를 설치하는 것은 컨테이너 경량화 및 자원 최적화 관점에 위배됨.
* **해결 방안:** 외부 패키지를 추가 설치하는 대신, 파이썬 내장 `os` 및 `open` 함수를 이용해 디렉토리 내 `.txt` 파일들을 메모리 효율적으로 단일 임시 마스터 파일(`merged_guidelines.tmp`)로 직렬화 병합한 후, 가볍고 무결한 `TextLoader` 스트림으로 처리하도록 인프라 파이프라인 우회 설계. 의존성 제로화 달성.

### 🔥 Issue 2: 초경량 모델(Gemma:2b)의 지시어 통제(Instruction Following) 실패 및 라우팅 오탐
* **현상:** 시스템 레이턴시 단축을 위해 1차 라우터로 `Gemma:2b`를 배치했으나, 모든 질의가 `SECURITY` 큐로 강제 인입되거나 RAG 문맥 내 핵심 데이터(90일 주기)를 인지하지 못하고 답변을 드랍하는 논리적 오작동 발생.
* **원인 분석 (정밀 로그 분석):**
  ```bash
  [⚙️ 인프라 라우터 로깅] 분류 결과: SECURITY (원문: GENERAL. The question is about the weather, not security.)
