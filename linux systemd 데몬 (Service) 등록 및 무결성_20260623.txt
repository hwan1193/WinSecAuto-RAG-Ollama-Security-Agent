linux systemd 데몬 (Service) 등록 및 무결성 검증

RAG 파이프라인 스크립트를 일회성 터미널 세션에 의존하지 않고, 리눅스 커널 수준에서 제어하고 운영하기 

### 1) 서비스 유닛 파일 생성

' /etc/systemd/system/security-rag.service' 경로에 백그라운드 인프라 설정을 덤프했습니다.

===================================================================
``` Ini, TOML

[Unit]
Description=Ollama FAISS Security RAG Agent Service
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/root
ExecStart=/usr/bin/python3 /root/Guidelines.py
Restart=on-failure

[Install]
WantedBy=multi-user.target
======================================================================

2) 서비스 활성화 및 시스템 등록

# systemd 매니저 구성 로드

systemctl daemon-reload

# 서비스 즉시 격발

systemctl start security-rag.service

# 부팅 시 자동 실행 활성화 (심볼릭 링크 생성 검증)

systemctl enable security-rag.service

# Created symlink /etc/systemd/system/multi-user.target.wants/security-rag.service → /etc/systemd/system/security-rag.service.

3) 서비스 구동 상태 및 실행 로그 검증 (systemctl status)

엔진 가동 결과, 파이썬 프로세스가 정상적으로 백엔드 호스트에서 컨텍스트를 연산하고 Success 코드로 안전하게 종료되었음을 확인했습니다.
